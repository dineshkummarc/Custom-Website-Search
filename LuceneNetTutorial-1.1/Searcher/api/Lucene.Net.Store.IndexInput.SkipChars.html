Expert Similar to ReadChars but does not do any conversion operations on the bytes it is reading in. It still has to invoke ReadByte just as ReadChars does, but it does not need a buffer to store anything and it does not have to do any of the bitwise operations, since we don't actually care what is in the byte except to determine how many more bytes to read
