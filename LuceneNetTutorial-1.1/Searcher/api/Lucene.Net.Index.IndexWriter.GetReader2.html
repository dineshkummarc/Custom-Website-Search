IndexWriter.GetReader(Int32) Method

Expert: like GetReader, except you can specify which termInfosIndexDivisor should be used for any newly opened readers.

public virtual IndexReader GetReader(
   int termInfosIndexDivisor
);

Parameters

termInfosIndexDivisor
Subsambles which indexed terms are loaded into RAM. This has the same effect as TermIndexInterval except that setting must be done at indexing time while this setting can be set per reader. When set to N, then one in every N*termIndexInterval terms in the index is loaded into memory. By setting this to a value > 1 you can reduce memory usage, at the expense of higher latency when loading a TermInfo. The default value is 1. Set this to -1 to skip loading the terms index entirely.
